## 2) 路线 URI：不要用 https://uri.amap.com/navigation（严重）

多方实测 + 高德文档：`https://uri.amap.com/navigation?...` 在 WebView 里**常停留在 H5**，不能可靠唤起原生 App。

**请统一改为：**
```
amapuri://route/plan/?sid=&slat={sLat}&slon={sLng}&sname={sName}&did=&dlat={dLat}&dlon={dLng}&dname={dName}&dev=0&t=0
```
`t`: 0驾车 1公交 2步行 3骑行

`view` 仍可用：`androidamap://viewMap?...`
单点导航仍可用：`androidamap://navi?...`（终点，起点取 GPS）

App 白名单请改为：`androidamap://`、`amapuri://`（**去掉** `https://uri.amap.com/` 作为主路径；若保留仅作未装 App 的 WebView 降级）

## 3) 坐标系：route.html 当前是 OSM+OSRM（WGS-84）

现有 `route.html` 用 OSRM + Leaflet OSM 瓦片，坐标是 **WGS-84**。
传给高德 URI 时若 `dev=0`（GCJ-02）会有**偏移**。

真导航场景（华亭嘉园→天安门）请：
- Canvas 页继续用高德瓦片坐标（已是 GCJ-02）✓
- 若用 OSRM，需 WGS→GCJ 转换，或改用高德路径 API
- 在文档标明每个 Canvas 页的坐标系

## 5) 触摸冲突（Canvas 页）

Leaflet 的 click/drag 区分不适用。Canvas 页 pan/zoom 与「点 marker 跳转」会冲突。
**推荐**：首版用**固定底部按钮**（不依赖点地图），避免与 ± 缩放抢触摸。
